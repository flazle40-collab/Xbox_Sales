# Xbox_Sales
Construção de Dashboard de Vendas
# Como Organizar um projeto de Dashboard?
É fundamental antes de iniciar o projeto manter a organização desde o momento da fase da ideação, implementação e manutenção. Uma maneira interessante
é o método A-B-C-D-E, que seriam abas específicas para cada etapa de construção.

A - Assets (recursos: imagens, vídeos, gifs, paletas de cores)

B - Bases (Dados para gerar a dashboard principal ou outras medidas)

C - Cálculos (Meio de campo que precessa dados para transformar em informações)

D - Dasboard (Painel visual com todos os elementos visuais que respondam as principais perguntas do négocio)

E - Extras (Informações relevantes que possam agregar ao Dashboard)

# Etapas
1- Entender a base de dados é essencial para um bom dashboard, conhecer cada campo
de cada tabela, ou seja, classificar se é relevante ou não para análises

2 - Transformar dados em informações (Responder uma pergunta de negócio, atráves de uma análise específica)

3 - Importância da Data View (Transformar algo complexo em um visual atrativo e simplificada. Utilizando formas, cores, e aparência, para cada situação) Dica de leitura: Não me faça pensar!

4 - Gráficos Dinâmicos e Filtros Visuais

5 - Organização e Estilização da Área de Dashboard, que deve ser o mais claro e sem poluição visual

6 - Trabalhando com Big Numbers, são números relevantes que devem ser apresentados em Cards

7 - Info Context, quando os dados foram atualizados? Qual o periódo analisado? Essas informações devem constar no Dashboard

8 - Toques Finais, deverá ser alinhada com os gestores e tomadores de decisão
